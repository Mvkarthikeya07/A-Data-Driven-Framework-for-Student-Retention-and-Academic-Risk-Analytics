## 🔭 Project Overview

Educational institutions struggle to detect at-risk students before academic outcomes deteriorate — traditional monitoring relies on fragmented records, manual supervision, and reactive intervention that arrives too late.

This framework introduces a **proactive academic intelligence system** that analyzes four student indicators — attendance, academic scores, behavioral engagement, and study effort — to classify each student into a risk tier and surface institution-level patterns through an interactive analytics dashboard.

**The system addresses three core institutional needs:**

- **Early risk detection** — identify students at High / Medium / Low academic risk before semester outcomes are locked in
- **Institutional analytics** — aggregate risk distribution, retention rates, and attendance trends across 10,000-student populations
- **Decision support** — give academic administrators a visual, data-backed foundation for targeted intervention

## 📊 Dataset

| Property | Value |
|---|---|
| **Total Records** | 10,000 student profiles |
| **Generation** | Synthetic, rule-consistent (`generate_dataset.py`, seed=42) |
| **Features** | 4 academic indicators + 1 computed target |
| **Retention Rate** | 21.75% (students meeting all passing thresholds) |
| **Avg Attendance** | 69.82% |
| **Avg Internal Marks** | 64.50 / 100 |
| **Avg Behavior Score** | 5.0 / 9 |

> **Note on synthetic data:** The dataset is programmatically generated from explicit academic rules rather than collected from real students. This was a deliberate choice to ensure privacy compliance and reproducibility. The rules themselves are grounded in commonly accepted academic performance thresholds.

### Feature Schema

| Feature | Range | Description |
|---|---|---|
| `attendance` | 40 – 100% | Student attendance percentage across the semester |
| `internal_marks` | 30 – 100 | Cumulative internal academic assessment score |
| `behavior_score` | 1 – 9 | Classroom engagement and behavioral rating (1 = worst, 9 = best) |
| `study_hours` | 0.5 – 6.0 hrs | Self-reported daily study effort |
| `final_result` | 0 / 1 | Academic outcome: 1 = Pass, 0 = Fail (computed target) |
| `risk_level` | High / Medium / Low | Multi-class risk classification label |

### Risk Band Distribution (10,000 students)

| Risk Level | Count | Percentage | Avg Attendance | Avg Marks | Avg Behavior |
|---|---|---|---|---|---|
| 🔴 High Risk | 5,845 | 58.5% | 62.8% | 59.3 | 4.3 |
| 🟠 Medium Risk | 2,238 | 22.4% | 73.6% | 65.4 | 6.0 |
| 🟢 Low Risk | 1,917 | 19.2% | 87.0% | 79.2 | 6.0 |

### Risk Classification Logic (from `generate_dataset.py`)

```python
# HIGH RISK — any of:
behavior_score <= 2                    # Severe engagement issues
attendance < 60                        # Critical attendance deficit
internal_marks < 45                    # Academic failure threshold

# MEDIUM RISK — borderline watchlist:
attendance < 75 OR internal_marks < 60

# LOW RISK — academically stable:
attendance >= 75 AND marks >= 60 AND behavior >= 3
```

## 🤖 Machine Learning Pipeline

### Architecture

```
student_data.csv  (10,000 records)
        │
        ▼
Feature Selection
  [attendance, internal_marks, behavior_score]
        │
        ▼
StandardScaler()          ← Normalize to mean=0, std=1
        │
        ▼
RandomForestClassifier    ← n_estimators=100, random_state=42
(n_estimators=100)
        │
        ▼
Stratified 80/20 Split    ← Train: 8,000 | Test: 2,000
        │
        ▼
model.pkl + scaler.pkl    ← Joblib serialization
        │
        ▼
Flask dashboard           ← Aggregated KPIs + risk table
```

**Three features are used for training** — attendance, internal_marks, and behavior_score — matching the same three indicators used in the risk classification rules that generated the labels.

### Understanding the Perfect Scores

The perfect 100% accuracy for tree-based models is **expected and correct** for this dataset — not a sign of overfitting. Here's why:

The risk labels in `student_data.csv` were generated by deterministic, axis-aligned threshold rules in `generate_dataset.py`:

```python
High Risk:   behavior_score <= 2  OR  attendance < 60  OR  internal_marks < 45
Medium Risk: attendance < 75  OR  internal_marks < 60
Low Risk:    all remaining
```

Decision Trees, Random Forests, and ensemble tree models **learn exactly these threshold splits** from the three numeric features. There is no noise, no ambiguity, and no overlapping class regions — the mapping from features to labels is mathematically perfect by construction.

**This is architecturally significant** — it demonstrates that the ML model and the rule-engine are fully aligned. When deployed on real student data, the model generalizes from the rule logic it learned, applying it to new records without requiring the rule thresholds to be hard-coded in production.

**Logistic Regression's lower accuracy (73.4%)** is expected: the class boundaries in this dataset are non-linear threshold conditions, which a linear classifier cannot perfectly represent.

---

### Why Random Forest Was Deployed

| Factor | Random Forest | Decision Tree | Gradient Boosting |
|---|---|---|---|
| **Accuracy** | 100% | 100% | 100% |
| **Overfitting Resistance** | ✅ High (ensemble of 100 trees) | ❌ Prone to overfitting on new data | ✅ High |
| **Feature Importance** | ✅ Built-in, reliable | ⚠️ Single-tree estimates | ✅ Available |
| **Inference Speed** | ✅ Fast | ⚡ Fastest | 🔶 Slower |
| **Noise Robustness** | ✅ Strong (bagging) | ⚠️ Sensitive to noise | ✅ Strong |
| **Industry Standard** | ✅ Widely used in EdTech | ⚠️ Too simple for production | ✅ Strong |

Random Forest is the preferred choice because it provides the **same perfect accuracy as a single Decision Tree** while being significantly more robust to noise and generalization to unseen data distributions — critical when transitioning from synthetic to real institutional data.

## 📐 Feature Importance Analysis

The trained Random Forest model (`model.pkl`) assigns the following importance weights across the three input features:

| Feature | Importance Score | Interpretation |
|---|---|---|
| `attendance` | **0.4804 (48.0%)** | Dominant predictor — absenteeism is the strongest single signal of academic risk |
| `internal_marks` | **0.3204 (32.0%)** | Strong secondary predictor — continuous assessment performance |
| `behavior_score` | **0.1993 (19.9%)** | Supporting signal — engagement quality amplifies risk from poor attendance/marks |

**Key insight:** Attendance alone accounts for nearly half of all predictive power. An institution monitoring only exam results misses the most important early warning signal.

## 🖥️ Analytics Dashboard

The Flask dashboard (`app.py`) aggregates the 10,000-student dataset into institution-level metrics served on a single route (`/`).

### KPI Metrics Computed

| Metric | Computation | Value (Full Dataset) |
|---|---|---|
| Total Students | `len(df)` | 10,000 |
| Retention Rate | `mean(final_result) × 100` | 21.75% |
| Avg Attendance | `mean(attendance)` | 69.82% |
| High Risk Count | `count(risk_level == "High")` | 5,845 (58.5%) |
| Medium Risk Count | `count(risk_level == "Medium")` | 2,238 (22.4%) |
| Low Risk Count | `count(risk_level == "Low")` | 1,917 (19.2%) |

### Visual Components

- **Risk Distribution Chart** — doughnut/bar chart of Low / Medium / High student counts
- **Monthly Attendance Trend** — 12-month line chart derived from rolling index grouping
- **Student Risk Table** — sampled 25-student view with individual risk labels and academic scores

## 🚀 Quick Start

### Prerequisites

- Python 3.8 or higher
- pip package manager

### 1. Clone the Repository

```bash
git clone https://github.com/your-username/A-Data-Driven-Framework-for-Student-Retention-and-Academic-Risk-Analytics.git
cd A-Data-Driven-Framework-for-Student-Retention-and-Academic-Risk-Analytics
```

### 2. Install Dependencies

```bash
pip install -r requirements.txt
```

**Pinned dependencies:**
```
flask==3.0.0
pandas==2.1.4
numpy==1.26.2
scikit-learn==1.3.2
joblib==1.3.2
```

### 3. (Optional) Regenerate the Dataset

```bash
cd dataset && python generate_dataset.py
```

Outputs `student_data.csv` — 10,000 records with attendance, marks, behavior, study hours, final result, and risk level.

### 4. (Optional) Retrain the Model

```bash
cd model && python train_model.py
```

Outputs `model.pkl` and `scaler.pkl` to the `model/` directory.

### 5. Launch the Dashboard

```bash
python app.py
```

Navigate to `http://127.0.0.1:5000` — the institutional analytics dashboard loads immediately.
